**v4.14** — **`loaders.js` split + version single-source-of-truth release.** Drops the patch on merge of `claude/review-repo-structure-0xgu9` into main. No user-facing UI changes — pure architecture cleanup of two findings from a repo review on this branch. **Version SSOT (v4.13.1):** `package.json` `"version"` is now the sole source of truth for the user-facing version, the `dist/*` cache-bust query strings in `index.html`, and the `sw.js` `CACHE` constant. `build.mjs` reads `package.json`, injects `__APP_VERSION__` via esbuild's `define` for both the JS bundle and `sw.js`, and post-build rewrites the `?v=X.Y.Z` query strings on `index.html`. Bumping any of those manually is no longer required. CLAUDE.md rule #7 rewritten accordingly. Final layout: 7 sibling section modules (`home.js`, `schedule.js`, `standings.js`, `news.js`, `league.js`, `live.js`, plus the pre-existing `yesterday.js`) and a 6-module `stats/` subtree (`_shared.js`, `leaders.js`, `team.js`, `roster.js`, `player.js`, `compare.js`). Methodology now documented for future extractions. **Module graph + functions doc + sections doc updated to reflect the new layout.** Per the new CLAUDE.md rule #7, `sw.js` `CACHE` is no longer bumped manually — derived from the `package.json` version via the build, so it auto-bumps on every release that touches anything.

