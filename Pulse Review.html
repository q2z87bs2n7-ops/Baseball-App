Pulse · design review

Pulse: easy-win sweep

A targeted pass on the parts of Pulse that read inconsistent today, framed as low-risk CSS-only patches Claude Code can apply one at a time. The thesis: Pulse is supposed to be team-neutral, but the global --accent token is wired to #FF5910 (Mets orange), which leaks team color into ~10 places that the Pulse view inherits. Plus a few smaller inconsistencies in story badges, hype pills, and badge color use.

In scope

CSS token swaps and copy-level changes inside index.html only. No JS function rewrites, no DOM restructuring, no template changes to pulse-card-templates.js. Every change is reversible by reverting one line.

Out of scope

The four Pulse popup card variants (V1–V4 in pulse-card-templates.js) — those are a finished spec already in handoff. The polling/state logic. The audio system. Any refactor of showPlayerCard / showRBICard. Restructuring the :root variable system itself (would touch the whole app).

01

Add a Pulse-scoped neutral accent token

The single biggest source of "team color leaking into Pulse." Today, --accent at :root is #FF5910 (Mets orange) and never gets overridden inside #pulse. Every rule that reads var(--accent) picks that up.

Low risk XS

Rather than touch :root (which would ripple into Home/League/Schedule/Standings — high risk), we introduce a Pulse-only override that re-points --accent when the user is inside the Pulse section. Every existing var(--accent) read inside Pulse picks up the new value automatically. Zero JS, zero markup change.

index.html~line 396, append at end of /* ── League Pulse ── */ block
@@ index.html — inside <style>, after the existing #pulse rules @@  .pulse-feed-label .pulse-bolt { color: var(--accent); }
  #feedWrap { margin: 8px 0 0; padding: 14px 12px 24px; border: 1px solid var(--border); border-radius: var(--radius); }
+ /* Pulse is team-neutral. Re-point --accent only inside the Pulse view. */
+ #pulse {
+   --accent: #cfd3dc;          /* neutral light grey — same hue family as --text */
+   --accent-soft: rgba(255,255,255,0.08);
+   --accent-strong: #ffffff;
+ }
+ /* Carry the override to children that escape #pulse for layout (alert toasts use it for the live badge). */
+ /* If you want global #alertStack to stay neutral too, also add this — otherwise skip. */
+ /* #alertStack { --accent: #cfd3dc; } */
Why this is safe: CSS custom properties cascade. Adding --accent on #pulse only affects descendants of #pulse. The header's nav button "active" state, the standings header, calendar today-marker, settings panel labels — all live outside #pulse and keep their team-tinted accent. The only things this changes are: .pulse-feed-label .pulse-bolt, the side-rail count and time badges, story-dot active state, the news-card link, and the demo-empty-state hype pills (covered separately below).
Pick the neutral hex deliberately: #cfd3dc is a desaturated cool grey that sits between --text (#e8eaf0) and --muted (#9aa0a8). If you want a faint warm-cool tilt instead, #b8c2d6 reads slightly cooler and is still WCAG AA against --card. Avoid pure white — too loud against the navy chrome.
02

Reduce story-badge palette from 9 colors to 4

The story carousel currently rotates between badges in 9 distinct colors. Stacked, this reads as "every story is a different category" rather than a coherent feed.

Low risk S

Today the badges are: live (red), today (green), final (blue), yesterday (muted grey), onthisday (purple), upcoming (amber), season (gold), probables (light blue), highlight (orange). That's 9 distinct hues across what is functionally one component. Group them by semantic intent and use saturation/temperature to differentiate only where it matters: live state vs everything else.

03

Story tier backgrounds — keep tier1, mute the rest

Story cards use 4 tinted backgrounds (tier1 teal, tier2 green, tier3 blue, tier4 grey) plus a special red tint for .story-biginning. Stacked vertically, this reads like a Christmas tree.

Low risk XS

The tier system encodes editorial importance — that's worth keeping for tier1 (the "lead story"). For tier2/3/4, drop the tinted background and rely on the left border-color to keep the priority cue without the wash. .story-biginning can keep its red tint since it's a genuine alert state ("big inning happening right now").

08

Border-radius alignment in Pulse area

Five different radii are in play in Pulse: --radius (used by .story-card, #feedWrap), 10px (side rail containers, .upcoming-hero), 12px (.card), 8px (ticker games, badges), 0 (feed-items, except first/last). Some of this is intentional rhythm; some is drift.

Low risk XS

The cleanest narrow fix: pick one radius for "container card" and one for "row chip", then apply. Keep 0 on stacked feed items (intentional — they share borders to read as a list). Don't touch --radius's root definition (used app-wide).

Hard rules for the Claude Code handover

  1. Do not modify :root, --accent at the root level, the team theme system, or any JS function. CSS-only, scoped to #pulse and the explicit selectors named in each diff.
  2. Do not touch pulse-card-templates.js. The popup card variants are out of scope here.
  3. Apply changes one at a time, in the order above. After each change, reload, switch the team to at least two different colors (Mets / A's are good extremes), and confirm the change took effect only inside the Pulse view.
  4. If any change visually regresses — for example, a kicker becomes unreadable on its background — stop and flag. Don't try to fix by going further; revert and ask.
  5. For Finding 02 (badge collapse): the JS badgeMap at line 2484 stays untouched. Same keys, just different styling. If the diff produces a CSS validation warning about the long comma-separated selector, that's fine; modern browsers handle it.
  6. For Finding 04: the optional emoji-strip is a copy decision in the JS at line 2854. Skip it on first pass. Only do it if the user explicitly approves after seeing the visual.
  7. Bump the version string after step 8 lands. Suggested: v1.18.1-pulse-neutral.

Manual visual verification matrix

After all 8 changes are in, run this matrix once to confirm nothing leaked.

  1. Team-color leak check: Open Pulse with team = Mets. Then settings → switch to Athletics. The Pulse view's chrome should be visually identical in both states (only the per-game team dots and team logos differ). If anything else changed color, an --accent read got missed.
  2. LIVE-state check: with at least one game live, the LIVE badge in the story carousel and the "● LIVE" inside ticker games should still be red. (We deliberately kept this.)
  3. Empty-state check: visit Pulse before any game starts. The hype pills, the "TODAY · N UPCOMING GAMES" kicker, and the demo CTA should all be neutral grey. The "NEXT UP" hero card keeps its team-color gradient (intentional — it's identifying a specific matchup, not chrome).
  4. Other-section check: visit Home, League, Schedule, Standings, Stats. None of these should change. Their --accent still points at the team's primary at the :root level.
  5. Popup card check: trigger an HR or RBI popup (or use the demo trigger if one exists). The popup cards should be unchanged — they're rendered by pulse-card-templates.js with inline styles and don't read --accent.
